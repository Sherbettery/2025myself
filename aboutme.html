個人簡介和申請動機

我是2021年第一次申請WeHelp訓練營，那時候是英語研究所在學，因為修了系上的量化研究課程，使用Python和R等語言進行語言分析，所以對程式語言開始有興趣。\n 那時候覺得寫程式很有趣也很有成就感，希望將來的工作也可以跟寫程式有關，而網頁開發也聽說是比較容易入門軟體業的領域，看了彭彭的免費和付費課程還是做不出專案，於是就提交了bootcamp申請，希望可以畢業後直接銜接工作。\n 但可惜那時候沒有申請上，後來還陸續在系上機器學習與深度學習等跨領域的課程，也到資工系上了機器學習理論，希望能更了解這塊研究範疇。\n 最後在2023年下半年，我以深度學習分類模型研究語言中的幽默拿到畢業證書，而畢業前我也有申請另一家Ap***rks的訓練營但面試被刷掉。\n 在畢業前後，我也和同學一起看著Udemy網頁開發線上課程，每週討論遇到的問題，但這個小團體因為成員們的工作因素而不了了之。\n 我繼續緩慢的進行我的Udemy課程，一邊從論文的疲累中恢復。雖然一個人學習對我來說問題不大，但課程偏講解觀念，專案很少的情況下，有一點失去寫程式的手感和熱情。\n 2023年12月我找到了工作，我的線上課進度又更緩慢了。\n 到了2025年9月，我赫然發現帶觀念的部分其實不用看得那麼仔細，趕快進到最後的成果專案，有不懂的再回去複習，可能會比較快，所以進度一下子快了不少，這課程過了多年總算是來到了60%完成度。\n 我想著，再這樣下去，肯定會安逸於目前的工作而始終無法轉職成功，但我已經脫離學校了，我還能再次適應這種自律規範嚴格的團體中學習嗎？\n 但現在不申請，接下來的日子我肯定會一直想為什麼之前不申請，於是乎我現在開始打這份申請書了。

    Google AppSheet App 串接 Gemini API

    • 簡介：AppSheet是Google一個不需要寫程式，可以直接將Google Sheets當作資料庫，並做出簡易應用程式的服務，可以加入自己寫的程式做簡易的自動化功能。我用Google AppScript寫程式串接Gemini API並加到這個自動化的程序中，做出一款老師可以針對不同受眾（ex:家長、學生）和情境（ex:校外教學、段考）等用AI寫出適合的溝通文字的app。
    • 製作目的：我們公司有代理Google教育型產品，因為在討論學校老師什麼情境會需要AppSheet這產品，所以做了這個。
    • 應用程式：https://www.appsheet.com/start/b39310a2-c613-4c5b-a086-b47e3d95027a
    • 程式碼：https://script.google.com/d/1RnkaC7hQofKruoFdj4KHvCyYIwKK5nsnS1qvcWwt-rVYNjp6yctpePin/edit?usp=sharing

    串接 Google Slides API 與 Gemini API 的簡報翻譯專案

    • 程式碼（一個block一個block執行，無使用者介面）：https://github.com/Sherbettery/translate_slides/blob/main/pptTranslate.ipynb
    • 簡介：利用Slides API取得簡報上的每個文字方塊，丟到list中，送進Gemini API翻譯，再將文字方塊原本的文字帶換成翻譯後的文字，還可以保留原本的字型、大小、顏色、超連結等要素，如果是pptx也可以先轉換成Google Slides。
    • 製作目的：這個是我有時候要翻譯近百頁的英文產品簡報，自己翻太累了，於是寫了這專案。

如何看待人工智慧的發展對軟體工程師的影響？

人工智慧可以提供很多程式碼邏輯的想法，我覺得對於我這個新手來說，他可以提示你如何更簡潔的寫程式碼，當然，要小心他可能漏掉你特別注重的某個部分，但有一個提示總是好的，跟查網路一樣，多一寫想法輔助你。\n 至於取代人工的議題，確實在使用AI的過程中，他會越來越懂你的邏輯，可以更好的的輔助你，可能有些老手就開始排斥將作業交給其他人了。但我認為這種時候，新手工程師應該學會更聰明且努力的去學習這個普遍的工具背後的原理和使用訣竅是什麼，以及冷靜應對的情緒。\n 比如說，給AI一個問題，他通常會一次給很多個可以去嘗試的點，但是如果對每個方向的知識都很淺，只盲目的嘗試AI講的方法，那試起來會花很多時間，反而比查網路更讓人疲累，但如果有充實相關知識，能夠抓住一兩個點並引導AI方向，那有機會快速的解決問題。\n 另外關於轉不轉職，我也認為不能還沒嘗試就斷言甚至直接放棄。
